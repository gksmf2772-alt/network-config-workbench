현재 규칙은 크게 4종류입니다.

**1. 객체 매핑**
- 위치: 프로파일 탭 → `객체 매핑`
- 목적: 벤더별 객체 타입 이름을 맞춥니다.
- 예:
  - 기존 `static-route`
  - 신규 `static-route`
  - 또는 신규 장비에서 다른 타입으로 파싱될 때 같은 타입으로 보정
- 영향:
  - 객체 타입 비교
  - 객체 키 생성
  - 비교 요약의 객체 분류

**2. 동일 객체 기준**
- 위치: 프로파일 탭 → `동일 객체 기준`
- 목적: “어떤 객체끼리 같은 객체인가”를 정합니다.
- 옵션:
  - `객체 시작 라인`: 기본값. `port 1/1/1`, `neighbor 1.1.1.1` 같은 첫 객체 라인 기준
  - `description 값`: 객체 내부 description에서 ID 추출
  - `사용자 정규식`: 객체 내부 전체 라인에서 정규식으로 ID 추출
- 예:
  - 기존 `port 1/1/1`
  - 신규 `port 2/2/c15/1`
  - 둘 다 `description "## Ulsan-TOD-FE73 ##"`이면 같은 객체로 매칭 가능

**3. 라인 매핑**
- 위치: 프로파일 탭 → 설정 예제 규칙 → `선택 라인 매핑 추가`
- 목적: 문법은 다르지만 의미가 같은 라인을 같은 라인으로 봅니다.
- 예:
  - `no shutdown` ↔ `admin-state enable`
  - `mode access` ↔ `mode access`
- 사용법:
  - 기존 예제에서 전체 라인 선택
  - 신규 예제에서 대응되는 전체 라인 선택
  - `선택 라인 매핑 추가`
- 영향:
  - 비교 전에 신규 라인을 기존 라인 기준으로 치환해서 비교합니다.

**4. 라인 규칙**
- 위치: 프로파일 탭 → `기존 선택 규칙 추가`, `신규 선택 규칙 추가`
- 각 규칙 액션:
  - `동일`: 현재는 표시용 성격이 강합니다.
  - `추가됨`: 신규에만 있어도 차이로 보지 않음.
  - `사라짐`: 기존에만 있어도 차이로 보지 않음.
  - `무시`: 해당 라인은 비교에서 제외.
  - `필수 존재`: 선택한 전체 라인이 반대쪽에도 있어야 함.
  - `필수 변수`: 값은 달라도 되지만 같은 필드가 양쪽 객체에 있어야 함.

**필수 변수 예시**
- 규칙 선택: `authentication-key`
- 기존:
  - `authentication-key "AAA" hash2`
- 신규:
  - `authentication-key "BBB" hash2`
- 결과:
  - 값은 달라도 정상
- 신규에서 라인 자체가 없으면:
  - 요약에 사용자 입력 로그 표시
  - 예: `authentication 키가 없습니다.`

**요약 로그**
- 라인 규칙 하단의 `요약 로그(선택)` 입력칸
- 해당 규칙 위반 시 비교 요약에 표시됩니다.
- 비워두면 기본 메시지가 표시됩니다.

**권장 사용 순서**
- 먼저 `동일 객체 기준` 설정
- 그 다음 `라인 매핑`으로 문법 차이 보정
- 마지막으로 `필수 변수`, `무시`, `추가됨/사라짐` 규칙 추가
- 저장 후 비교 탭에서 프로파일 적용 → 비교 실행

기존 config에만 존재
신규 config에만 존재

기존 config와 신규 config에 동일해야 하는 값(단 변경 가능한 값)
 - ip, route, bgp neighbor

기존 config와 신규 config에 동일하지 않아도 되는 값(단 변경 가능한 값)
 - port, lag, interface name

기존 및 신규 config에 존재해야 하는 설정
기존에 존재하면 신규에도 존재해야 하는 설정(단 기존에 없으면 신규에도 없어도 됨)
기존 config에서 동일한 기능이지만 문법 또는 라인 수가 바뀐 신규 설정
예외 케이스 설정(예- 기존에는 tag 700, 701이었지만 신규는 다 tag 700으로 하기로 합의)
